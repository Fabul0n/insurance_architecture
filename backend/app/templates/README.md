# Шаблон договора `insurance_blank.docx`

Положите ваш исходный бланк в файл:

- `backend/app/templates/insurance_blank.docx`

Бэкенд заполняет **только текстовые метки** в документе и не перестраивает его структуру.

## Метки для вставки в DOCX

Используйте любой из двух стилей (оба поддерживаются).

### Стиль 1 (рекомендуемый)

- `{{CONTRACT_NUMBER}}` — номер договора
- `{{CONTRACT_DATE}}` — дата договора (дд.мм.гггг)
- `{{POLICYHOLDER_FULL_NAME}}` — ФИО страхователя
- `{{POLICYHOLDER_PASSPORT}}` — паспорт страхователя
- `{{POLICYHOLDER_BIRTH_DATE}}` — дата рождения страхователя (дд.мм.гггг)
- `{{POLICYHOLDER_EMAIL}}` — email страхователя
- `{{WORKPLACE}}` — место работы
- `{{INSURANCE_OBJECT}}` — объект страхования
- `{{INSURANCE_PERIOD_MONTHS}}` — срок страхования (мес.)
- `{{INSURANCE_CASES}}` — страховые случаи (строка)
- `{{PAYOUT_AMOUNT}}` — сумма выплаты
- `{{INSURER_FULL_NAME}}` — ФИО страховщика

### Стиль 2 (альтернативный)

- `<НОМЕР_ДОГОВОРА>`
- `<ДАТА_ДОГОВОРА>`
- `<ФИО_СТРАХОВАТЕЛЯ>`
- `<ПАСПОРТ_СТРАХОВАТЕЛЯ>`
- `<ДАТА_РОЖДЕНИЯ_СТРАХОВАТЕЛЯ>`
- `<EMAIL_СТРАХОВАТЕЛЯ>`
- `<МЕСТО_РАБОТЫ>`
- `<ОБЪЕКТ_СТРАХОВАНИЯ>`
- `<СРОК_СТРАХОВАНИЯ_МЕС>`
- `<СТРАХОВЫЕ_СЛУЧАИ>`
- `<СУММА_ВЫПЛАТЫ>`
- `<ФИО_СТРАХОВЩИКА>`

## Важно

- Вставляйте метку как цельную строку (не разбивайте вручную на части).
- Если метка встречается несколько раз, она будет заменена во всех местах.
- Параметр в конфиге: `CONTRACT_TEMPLATE_PATH` (по умолчанию `app/templates/insurance_blank.docx`).
